# CS 305 – Secure Software Development Portfolio

## Client: Artemis Financial
Artemis Financial is a financial services company that required a review of its software security practices. The goal of this project was to identify vulnerabilities and apply secure coding techniques to protect sensitive financial data.

## Project Summary
This project focused on identifying security vulnerabilities, improving encryption practices, and ensuring secure communication within the application.

## Reflection
One area I performed well in was identifying software vulnerabilities using both manual review and automated tools. Secure coding is critical because vulnerabilities can expose sensitive data, damage trust, and create legal and financial risks.

The most challenging part of the vulnerability assessment was interpreting scan results and prioritizing risks. However, this process improved my ability to analyze security issues critically.

I increased layers of security by implementing HTTPS, encryption, and hashing techniques. In the future, I would continue using vulnerability scanners and OWASP guidelines to assess and mitigate security risks.

After refactoring the code, I tested functionality and re-ran security scans to ensure no new vulnerabilities were introduced.

Tools used in this project included OWASP Dependency-Check, Java Keytool, and secure coding best practices. These tools will be valuable in future coursework and professional projects.

This project demonstrates my ability to assess vulnerabilities, apply secure coding practices, and document security improvements for professional use.
